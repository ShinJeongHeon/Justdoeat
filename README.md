# 🍽️ Just do eat — 자동 점심 메뉴 추천

> **"인생은 선택의 연속. 점심 메뉴만큼은 고민하지 마세요. Just do eat!"**
>
> 내 위치 주변 식당을 바탕으로, 예산·식성·알레르기·일정·과거 피드백을
> 모두 고려해 **매일 점심 TOP 3를 이유와 함께 추천**하는 서비스입니다.
> 1차 타깃은 직장인, 위치를 자유롭게 바꿀 수 있어 출장자·여행자 등 누구나 확장 사용 가능.

## 실행 방법

`index.html`을 브라우저에서 열면 바로 동작합니다. 별도 서버·설치 불필요.
데이터는 브라우저 localStorage에만 저장됩니다.
(지도는 OpenStreetMap 타일을 사용하므로 인터넷 연결이 필요합니다. 오프라인이면 지도만 빠지고 나머지는 정상 동작.)

## 기능 명세

### 추천

| 기능 | 구현 방식 |
|---|---|
| **오늘의 추천 TOP 3** | 무한 재추천 대신 매일 3곳을 순위·이유와 함께 제시 (선택 피로 방지) |
| 추천 이유 표시 | "⏱️ 점심 40분 — 빨리 나오는 곳", "🍻 저녁 회식 전이라 가볍게" 등 칩으로 근거 설명 |
| 매일 다른 메뉴 | 날짜 시드 랜덤 + 최근 2주 방문 감점 + 같은 카테고리 연속 방지 |
| 예산 | 1끼 예산 초과 식당은 후보에서 완전 제외 (하드 필터) |
| 알레르기 | 6대 원료(밀·계란·유제품·대두·견과류·갑각류) — 해당 식당 절대 추천 금지 |
| 식성 | 카테고리 ❤️선호/✕기피 + 매운맛 3단계 |
| 인원 | 5명 이상이면 단체석 식당만 |
| 일정(캘린더) | 점심 가능 시간 자동 계산 → 왕복 이동+식사 시간이 안 되는 곳 제외. 저녁 회식이면 가벼운 메뉴, 점심 약속이 있으면 추천 쉬어감 |
| 날씨 | **Open-Meteo 기상 API로 자동 감지** (수동 입력 불필요, 위치 이동 시 재조회) — 비 = 300m 이내, 무더위 = 시원한 메뉴, 한파 = 뜨끈한 국물 우선. 수동 오버라이드 가능 |
| **"왜 이 집이에요?"** | 카드에서 점수 구성(가감점 전체)과 **반경 내 같은 종류 경쟁 식당과의 비교**("○○보다 +12점, ○○의 감점: 3일 전 이미 다녀옴 −30")를 투명하게 공개 |

### 실시간 커뮤니티 (사용자가 많아질수록 정확해짐)

- **실시간 혼잡도**: "오늘 이 앱에서 N명 선택" — 주변 사용자들이 '이걸로 결정'한 수를 집계해 🔴붐빔/🟠보통/🟢여유 표시. 점심시간이 짧은 날 붐비는 집은 순위 하락
- **오늘의 인기 식당**: **내 위치의 탐색 반경 안에서** 매일 가장 많이 선택된 식당 TOP 3 표시 + 1위 배지 (위치를 옮기면 그 동네 기준으로 바뀜)
- **당일 인증 리뷰**: 리뷰는 **그날 결정한 식당에 대해 그날만** 작성 가능 → 방문하지 않은 사람의 허위 리뷰가 구조적으로 차단됨 (리뷰 신뢰성 확보). "왜 이 집?" 패널에서 다른 사용자의 최근 인증 리뷰 노출
- 프로토타입에서는 다른 사용자를 시뮬레이션으로 대체 (실서비스: 서버 집계)

### 위치 & 지도

- **지도(Leaflet + OpenStreetMap)**: 내 위치(🏢), 탐색 반경 원, 식당 마커 표시. TOP 3는 주황 강조, 반경 밖은 흐리게
- **탐색 반경 설정**: 200m~1,500m 슬라이더 — 반경 밖 식당은 추천 제외
- **위치 변경(출장 모드)**: 지도 클릭 또는 프리셋(시청/강남/판교/여의도/부산)으로 이동, 위치 이름 저장
- **식당 정보 링크**: 각 추천 카드와 지도 팝업에서 🔗 클릭 시 식당 정보 페이지(네이버 지도)로 이동 — 실서비스에선 식당 공식 홈페이지 URL DB 연동

### 정보 제공

- **칼로리·영양소**: 1인분 kcal, 단백질/탄수화물/지방(g) — 식당 제공 정보가 있으면 그것을, 없으면 표준 영양정보 추정치로 표시(출처 라벨 구분)
- **원산지**: 주요 식재료 원산지 표기 (예: "돼지고기 국내산 · 김치 국내산")

### 기록 & 학습

- **식사 달력**: "이걸로 결정"을 누르면 그날 먹은 식당·메뉴가 달력에 자동 기록 (이모지 + 피드백 상태 점)
- **피드백 학습**: 달력에서 날짜 클릭 → 👍 맛있어요(가산점) / 👎 별로예요(감점) / ⛔ 다신 안 가요(영구 제외)
- **월 점심 예산 관리**: 결정할 때마다 지출 누적, 예산이 빠듯해지면 저렴한 곳 가중치 자동 상승
- **월간 리포트**: 달력에서 보는 달의 요약 — 기록한 끼니 수, 총 지출(예산 대비 %), 평균 한 끼 가격,
  평균 칼로리, 점심 산책 거리(왕복 도보 합계), 👍/👎 수와 새로 도전한 집 수,
  카테고리별 "무엇을 먹었나" 분포, 이 달의 최애 식당 🥇🥈🥉.
  기록이 없으면 예시 기록으로 미리보기 가능

## 추천 엔진 로직

```
1) 하드 필터 (하나라도 걸리면 후보 제외)
   알레르기 충돌 · 예산 초과 · 탐색 반경 밖 · ⛔ 블랙리스트
   · 단체석 없음(5인↑) · 못 먹는 매운맛 · 점심 가능 시간 부족

2) 소프트 점수 (기본 50점 + 날짜 시드 랜덤 0~10)
   +15 좋아하는 카테고리        −25 기피 카테고리
   +18×👍                       −25×👎
   +12 날씨 적합                −(최근 방문일수록 최대 −36)
   +14 회식날 가벼운 메뉴        −15 직전과 같은 카테고리
   +10 월 예산 빠듯 시 저렴한 집

3) 상위 3곳 = 오늘의 추천 TOP 3 (이유 칩과 함께)
```

## 프로덕션 전환 시 연동 포인트

| 프로토타입 | 실서비스 |
|---|---|
| 예시 식당 24곳 (위치 기준 상대 배치) | 카카오/네이버 지역 API로 실제 주변 식당 실시간 수집 |
| 수동 일정 입력 | Google Calendar API (`calendar.readonly` 스코프, 당일 이벤트 조회) |
| 날씨: Open-Meteo (이미 실연동) | 기상청 단기예보 API로 정밀도 향상 가능 |
| 혼잡도·인기·리뷰: 시뮬레이션 | 실제 사용자 "결정" 이벤트 서버 집계 (반경별 실시간 카운트) |
| 표준 영양정보 하드코딩 | 식약처 식품영양성분 DB + 식당 제공 정보 우선 |
| 네이버 지도 검색 링크 | 식당 공식 홈페이지/예약 페이지 URL DB |

## 향후 로드맵 제안 (추가 기능 아이디어)

1. **팀 점심 모드** — 링크 공유로 팀원 취향·알레르기 합집합 추천 + 투표
2. **슬랙/카톡 봇** — 매일 11시 TOP 3 자동 발송.
   *DAU 우려에 대한 설계 원칙*: 봇은 유입 채널이고, 결정·피드백·리뷰·달력은 앱 딥링크로
   되돌아오게 설계 (봇에서 "결정" 탭 → 앱의 기록/리뷰 화면). 북극성 지표는 앱 오픈 수가
   아니라 **추천 수락률·기록률·리뷰 작성률**로 정의 — 서비스의 가치가 "선택 제거"이므로
   앱 체류시간을 늘리는 것은 오히려 가치 훼손
3. **타이밍 알림** — 혼잡도 기반 "11:40에 출발하면 웨이팅 없이 먹어요" 푸시
4. **건강 목표 모드** — 주간 칼로리/나트륨 목표 설정, 어제 과식했으면 오늘 가볍게 (다이어트 직장인)
5. **결제 연동 자동 기록** — 카드/페이 결제 내역으로 식사 기록·지출 자동 채움 (수동 입력 제로)
6. **탐험 슬라이더** — "안 가본 집 도전 ↔ 검증된 집만" 성향 조절, 새로 오픈한 식당 알림
7. **구내식당 연동** — 오늘 구내식당 메뉴를 0순위 후보로 비교 ("오늘은 나가서 먹을 가치가 있는 날")

(월말 리포트는 구현 완료 → 위 "기록 & 학습" 참고)
