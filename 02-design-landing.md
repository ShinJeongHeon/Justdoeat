# 디자인 스펙 — Just do eat 랜딩페이지

> 근거 문서: `01-prd-landing.md` (섹션 구조·범위) · `index.html` (디자인 시스템 v2 원본)
> 산출물: 의존성 없는 단일 `landing.html`

---

## 1. 디자인 컨셉

**"랜딩이 곧 데모다."**

이 서비스의 가치는 "선택 제거"이므로, 페이지가 말로 설득하는 순간 가치와 모순된다.
따라서 랜딩의 모든 시각 언어는 하나의 경험을 향한다 — **방문자가 조건 칩을 누르고,
추천이 눈앞에서 다시 계산되어 결론이 바뀌는 것을 직접 본다.**

- 정적인 부분은 제품(v2 시스템)과 같은 얼굴로 조용하게.
- 움직이는 부분은 단 하나의 시퀀스("결정의 순간")에 집중 투자.
- 3D는 장식이 아니라 **"접시가 바뀐다 = 결론이 바뀐다"** 를 표현하는 서사 장치로만 사용.

### 자기검증 — 피한 디폴트

| 흔한 AI 랜딩 디폴트 | 이 페이지의 선택 |
|---|---|
| 크림+테라코타, 이모지 장식, 그라데이션 텍스트 | v2 팔레트 고정(쌀밥 화이트·정장 네이비·고추장 레드), 장식 이모지 금지 |
| 히어로 = 큰 숫자 + 스탯 3개 + 스크린샷 | 히어로 = **살아있는 미니 데모** (추천 엔진 축소판) |
| 기능 아이콘 3열 그리드 × N줄 | 가치 3축만, 나머지 기능은 텍스트 칩 한 줄로 강등 |
| 떠다니는 도형·파티클·패럴랙스 남발 | 모션은 "결정의 순간" 시퀀스 1곳 + 은은한 스크롤 리빌만 |
| WebGL 3D 오브젝트 | CSS 3D(접시 플립·카드 틸트)만 — 의존성 0, 성능 예산 내 |

## 2. 디자인 원칙 (v2 계승 + 랜딩 확장)

1. **얼굴 일치**: 랜딩과 앱(`index.html`)은 같은 제품의 두 화면. 토큰·로고·어휘를 그대로 복사한다.
2. **어휘 일관**: CTA는 처음부터 끝까지 "지금 추천받기" 하나. 앱 어휘("이걸로 결정", "오늘 기록됨", "식사 기록")를 데모 안에서 그대로 사용.
3. **모션은 의미가 있을 때만**: 모든 애니메이션은 "추천이 다시 계산됐다"를 보여주거나, 읽기 순서를 안내하거나(스크롤 리빌), 조작 피드백(호버/프레스)일 것. 분위기용 상시 루프 애니메이션 금지.
4. **금지 목록**(v2에서 승계): 장식 이모지(음식 이모지는 접시 아바타로만), 포스터풍 레이아웃, 그라데이션 텍스트, radius 10~16px 범위 밖의 라운딩, 강한 그림자.

## 3. 토큰 시스템

### 3.1 색 — v2 그대로 (신규 색 추가 없음)

| 토큰 | 값 | 랜딩에서의 역할 |
|---|---|---|
| `--bg` | `#F2F3EF` | 페이지 배경 (쌀밥 화이트) |
| `--surface` | `#FFFFFF` | 카드·데모 패널 |
| `--ink` | `#1D2A38` | 본문·헤드라인·**마무리 CTA 밴드 배경(반전)** |
| `--sub` | `#5D6975` | 보조 텍스트 |
| `--line` / `--line2` | `#E4E6E0` / `#CFD3CA` | 보더 |
| `--brand` / `--brand-deep` | `#E0431F` / `#B5300F` | CTA·헤드라인 강조어·스푼체크 |
| `--brand-soft` | `#FCEBE4` | 칩 활성 배경 |
| `--ok` / `--ok-soft` | `#1E7A4C` / `#E3F1E8` | 가점·긍정 칩 |
| `--warn` | `#AF7712` | 혼잡 등 주의 데이터 |
| `--navy` / `--navy-soft` | `#31567F` / `#E9EEF5` | 정보 칩·포커스 링 |

유일한 랜딩 전용 확장: **[8] 마무리 밴드에서 `--ink`를 배경으로 반전 사용** (텍스트 `#FFF`, CTA는 `--brand`).
새 색을 만들지 않고 페이지에 "끝"을 부여하는 장치.

### 3.2 타입 스케일 — 랜딩은 앱보다 한 단계 크게 (도구가 아니라 읽는 페이지)

| 역할 | 서체 | 크기 | 비고 |
|---|---|---|---|
| Display XL (히어로 헤드라인) | Black Han Sans | `clamp(42px, 8.5vw, 74px)` | line-height 1.08, letter-spacing -.005em |
| Display L (섹션 헤드) | Black Han Sans | `clamp(26px, 5vw, 40px)` | |
| Display M (공감 한 줄·CTA) | Black Han Sans | `clamp(22px, 4vw, 32px)` | |
| 아이브로우 | Pretendard 800 | 12px | letter-spacing .14em, `--sub` 또는 `--brand` |
| 본문 | Pretendard 400/600 | 15.5px | line-height 1.7, `word-break: keep-all` |
| 데모 내부 | Pretendard | 앱과 동일 스케일(11.5~14px) | 데모는 "실물 축소판"이므로 앱 크기 유지 |
| 숫자·데이터 | Pretendard | — | `font-variant-numeric: tabular-nums` |

Black Han Sans는 단일 웨이트이므로 강조는 굵기가 아니라 **색**으로: 헤드라인 강조어("바로 결정")만 `--brand`.

### 3.3 간격·형태

- 섹션 세로 리듬: 모바일 72px / 데스크톱 112px (`--space-section`)
- 콘텐츠 최대폭: 1080px (앱의 1200px보다 좁게 — 읽기 폭 우선), 히어로만 1200px
- radius: 카드 14~16px, 버튼 10~12px, 칩 999px (v2 동일)
- 그림자: `--shadow` v2 값 그대로. 데모 카드에만 호버 시 `0 12px 32px rgba(29,42,56,.10)` 한 단계 추가

### 3.4 모션 토큰

| 토큰 | 값 | 용도 |
|---|---|---|
| `--dur-micro` | 150ms | 호버·프레스·칩 토글 피드백 |
| `--dur-reveal` | 450ms | 스크롤 리빌·히어로 등장 (앱 `rise`와 동일) |
| `--dur-flip` | 650ms | 접시 3D 플립 |
| `--ease-out` | `cubic-bezier(.2, .7, .3, 1)` | 기본 (앱과 동일) |
| `--ease-flip` | `cubic-bezier(.35, 0, .2, 1)` | 3D 플립 전용 |
| stagger | 80ms | 히어로 등장 시차 |

## 5. 섹션별 디자인 스펙

### [1] 히어로 — 살아있는 미니 데모

- **왼쪽(카피)**: 아이브로우 `JUST DO EAT`(스푼체크 마크 + 워드마크) → 헤드라인 `오늘 점심,` / `바로 결정.`("바로 결정"만 `--brand`) → 서브 카피 1문장 → 대형 CTA `지금 추천받기`(앱 `button.decide` 스타일, 스푼체크 아이콘 포함, `index.html`로 링크).
- **오른쪽(데모)**: 앱 히어로 카드의 **실물 축소판**. 구성 —
  - 상단: 조건 칩 4개 — `비 오는 날` `예산 절약` `매운 게 당김` `혼잡 회피` (토글 가능, 활성 시 `--brand-soft` 배경 + `--brand-deep` 텍스트)
  - 중단: 1위 카드 = 접시(`.plate` 모티프, 음식 이모지 아바타) + 식당명(Black Han Sans) + 이유 칩 2~3개
  - 하단: 2·3위 소형 행 + 안내 문구 `조건을 눌러 보세요 — 추천이 다시 계산됩니다`
  - 데이터: 고정 식당 풀 6~8곳, 인라인 JS로 점수 재계산 (PRD 7.2-1)
- **등장 모션**: 페이지 로드 시 아이브로우 → 헤드라인 → 서브 → 데모 카드 → CTA 순 80ms 시차 rise(10px↑, `--dur-reveal`). 1회만.
- **3D 틸트**: 데모 카드에 포인터 추적 틸트 — `perspective: 900px`, 최대 ±3°. `(hover: hover) and (pointer: fine)`에서만, 터치·reduced-motion에서는 없음.

### [2] 공감 한 줄 — 타이포그래피 휴지(休止)

- 배경 `--bg` 그대로, 카드 없음. 넉넉한 여백 속 Display M 한 문장:
  `"인생은 선택의 연속. 점심만큼은 고민하지 마세요."`
- "점심만큼은"만 `--brand`. 장식·아이콘 없음. 페이지에서 유일하게 비어 있는 구간 — 데모의 밀도와 대비.

### [3] 작동 방식 3단계

- 실제 시간 순서(조건 → 추천 → 학습)이므로 번호 사용이 정당. `①②③`은 `--navy-soft` 원형 배지.
- 3열 카드(모바일 1열). 각 카드: 배지 + 제목(Pretendard 800) + 설명 2줄 + 해당 UI 조각 1개(조건 패널 칩·TOP3 미니 카드·피드백 버튼 등 — 앱 컴포넌트 축소 재현, 정적).
- 카피:
  1. **내 조건 그대로** — "예산·알레르기·일정·날씨를 한 번 설정하면 매일 자동 반영."
  2. **매일 TOP 3 + 이유** — "무한 리스트 대신 오늘의 결론. 왜 이 집인지 근거까지."
  3. **먹을수록 정확하게** — "결정 → 기록 → 피드백. 내 입맛을 학습합니다."

### [4] 킬러 피처 — "왜 이 집이에요?"

- 2컬럼(모바일 스택): 왼쪽 섹션 헤드 `왜 이 집이에요?` + 설명("점수 구성과 경쟁 식당 비교까지 전부 공개합니다. 근거 없는 추천은 하지 않아요."), 오른쪽 앱 `.why` 패널 재현(가점 `--ok` / 감점 `--brand-deep` / 합계 굵은 상단 보더 — 앱 스타일 그대로).
- 스크롤 진입 시 가감점 행이 위→아래 60ms 시차로 rise — "계산이 쌓이는" 느낌. 1회만.

### [5] 신뢰 장치 — 당일 인증 리뷰

- 짧고 강하게 한 밴드. 헤드: `그날 결정한 사람만, 그날만.`
- 칩 다이어그램: `[오늘 결정] → [오늘만 리뷰 가능] → [내일이면 마감]` (navy-soft 칩 + 화살표). "허위 리뷰가 구조적으로 불가능합니다" 한 줄.

### [6] 기록·리포트 미리보기

- 식사 달력 + 월간 리포트 카드 목업 2장. **정적 깊이 스택**: 두 카드를 살짝 겹치고 각각 ±1.5° 회전 — 이 페이지에서 유일하게 허용되는 정적 기울임(움직이지 않음).
- 카피: `먹은 만큼 데이터가 됩니다` + "식사 달력, 월간 리포트, 좋아요/별로/제외 피드백."

### [7] 나머지 기능 — 한 줄

- 텍스트 칩 한 줄: `혼잡도` `영양정보` `원산지` `출장 모드` `지도`. 아이콘 그리드 금지. 출장 모드 칩에만 짧은 보조 문구("낯선 동네에서도").

### [8] 마무리 CTA — 잉크 밴드

- 페이지 유일의 반전 구간: 배경 `--ink`, 대형 스푼체크(화이트) + `JUST DO EAT` 워드마크 + `고민은 여기까지.` + CTA `지금 추천받기`(`--brand`).
- 밴드 radius 없음(풀블리드), 내부 콘텐츠만 중앙 정렬. footer 텍스트(프로토타입 고지) 포함.

## 6. 시그니처 — "결정의 순간" 시퀀스

조건 칩을 토글하면 재계산 결과가 하나의 안무로 나타난다. **페이지에서 기억에 남을 단 하나의 장면.**

```
t=0        칩 토글: --brand-soft 채움 + 살짝 press (--dur-micro)
t=0~400ms  순위 변동 시 카드 FLIP 재배열
           (기존 위치→새 위치 transform 이동, --ease-out. 2·3위 행이
            1위 슬롯으로 "승격"하는 이동이 눈에 보이게)
t=200~850ms 1위 접시 3D 플립: rotateY 0→180°, perspective 900px,
           --ease-flip. 90° 시점에 뒷면(새 음식 아바타 + 새 식당명) 노출
t=850~1150ms 아이브로우의 스푼체크 체크 획이 stroke-dashoffset으로
           다시 그려짐 (300ms) — "결정 완료" 도장의 v2식 표현
```

- 총 1.2초 이내. 연타 시 마지막 입력 기준으로 안무 중단·재시작(끊겨도 어색하지 않게 transform 기반).
- 순위가 안 바뀌는 토글이면 FLIP 생략, 이유 칩 교체 + 체크 드로잉만.
- `prefers-reduced-motion`: 전 단계 생략, 결과 즉시 교체(내용 변화는 동일하게 전달).

## 7. 모션·3D 시스템 요약

| 트리거 | 효과 | 규칙 |
|---|---|---|
| 페이지 로드 | 히어로 5요소 stagger rise | 1회, 80ms 시차 |
| 스크롤 진입 | 섹션 rise (10px↑ + fade) | IntersectionObserver, 섹션당 1회, threshold 0.2 |
| 칩 토글 | 시그니처 시퀀스 (§6) | FLIP + 3D 플립 + 체크 드로잉 |
| 포인터 이동 (데모 카드) | 3D 틸트 ±3° | hover+fine pointer에서만 |
| 호버 (CTA) | 배경 `--brand-deep` + 스푼체크 체크 드로잉 | `--dur-micro` |
| 호버 (카드) | 2px 리프트 + 그림자 한 단계 | `--dur-micro` |
| 히어로 CTA 이탈 (모바일) | sticky 하단 CTA 슬라이드 업 | 1회 등장 후 유지 |

**하지 않는 것**: 상시 루프 애니메이션, 패럴랙스 배경, 스크롤 하이재킹, WebGL/캔버스, 커서 트레일.

**구현 제약**: 모든 모션은 `transform`/`opacity`만 사용(레이아웃 리플로우 금지). 3D는 CSS `perspective`/`rotateY`/`backface-visibility`만. JS는 인라인, 라이브러리 없음.

## 8. 반응형 규칙

| 브레이크포인트 | 변화 |
|---|---|
| ≥1020px | 히어로 2컬럼(카피 1fr + 데모 460px), 3단계 3열 |
| 640~1020px | 히어로 스택(카피 → 데모), 3단계 3열 유지 가능하면 유지 |
| <640px | 전부 1열, 히어로 카피 3줄 이내로 압축, sticky 하단 CTA 활성 |

- 데모 카드는 어느 폭에서도 잘리지 않게 `minmax(0, …)` + 내부 스크롤 금지(내용을 폭에 맞게 축소).
- 터치 타깃 최소 44px (조건 칩 포함).

## 9. 접근성·품질 기준

- 조건 칩: `role="button"` `tabindex="0"` `aria-pressed`, Enter/Space 지원 (앱과 동일 패턴).
- 포커스: `:focus-visible` 2px `--navy` 아웃라인 (v2 동일).
- 시퀀스 후 결과 변경은 `aria-live="polite"` 영역으로 낭독 (`"추천이 바뀌었어요: 김밥천국 본점"`).
- `prefers-reduced-motion: reduce` → 모든 transform 애니메이션 제거, 내용 즉시 반영.
- 색 대비: 본문 `--ink`/`--bg` AA 이상, `--sub`는 12px 이상 텍스트에만.
- 이미지 없음(SVG·이모지 아바타만) → LCP는 헤드라인 텍스트. 폰트는 앱과 동일 소스 + `display=swap`.

## 10. 검증 루틴 (v2 승계)

1. `node --check`로 인라인 JS 구문 확인 (스크립트 추출 후).
2. 헤드리스 Chrome 스크린샷 — 데스크톱(1280px)·모바일(390px). 최소 창폭 ~500px 이슈가 있으므로 좁은 뷰포트는 iframe 하니스로 캡처.
3. 확인 항목: 히어로에서 데모 상단 노출 여부(모바일), 시퀀스 전/후 캡처 비교, [8] 반전 밴드 대비, reduced-motion 모드 렌더.
4. Lighthouse 모바일 성능 ≥ 90 (PRD KR4).

## 11. v3 재편 — "단조로움" 피드백 반영 (2026-07-08)

**피드백**: 디자인·설명이 단조롭다. 마음을 사로잡아 써보고 싶게 만들 것.
서비스 미출시 상태이므로 출시 알림 신청(전화번호·이메일·직장 동네) 수단 필요.

**진단**: 모든 섹션이 "아이브로우 → 헤드라인 → 흰 카드"의 동일 리듬을 같은 배경에서 반복했고,
카피가 기능 설명형이라 감정을 건드리는 장면이 없었다. 토큰·서체·모션 시스템(§3)은 문제가 아니므로 유지.

### 변경 요약

| 항목 | v2 | v3 |
